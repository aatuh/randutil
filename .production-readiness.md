## Verdict

`randutil` is now on the release-readiness path for ordinary production use.
The concrete blockers from the prior assessment have been remediated locally:

- `PickByProbability` rejects `NaN` and infinite probabilities with
  `core.ErrInvalidProbability`.
- Full signed integer ranges can return `math.MinInt64` and 64-bit
  `math.MinInt` without a spurious `ErrResultOutOfRange`.
- Deterministic regression coverage was added for the fixed edge cases.
- Distribution non-finite parameter coverage now includes Bernoulli
  probability validation.
- Runtime and audit-tool dependencies were refreshed.
- CI action versions were refreshed, and generated SARIF is normalized before
  upload to avoid duplicate-tag validation failures.
- README, package docs, release docs, changelog, and benchmark evidence were
  updated for the release.

Practical rating after local remediation: **around 8/10 production readiness**
for ordinary application use, assuming the final local gate and remote GitHub
checks pass.

## Current Production Boundary

Use `randutil` for ordinary Go application needs such as tokens, random bytes,
UUIDs, ULIDs, NanoIDs, sampling, deterministic tests, and controlled
domain-separated streams.

Do not present it as:

- a FIPS-validated module;
- an audited cryptographic primitive library;
- a replacement for direct `crypto/rand.Reader` use in strict compliance
  contexts;
- a battle-tested dependency for high-risk security SDKs without additional
  review;
- a statistical package with externally validated scientific distributions.
